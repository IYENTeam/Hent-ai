Hent-ai Generation Flow

A technical overview of the complete image generation lifecycle and prompt architecture.

1. Overview

7 Total Images

The system generates exactly one base anchor image followed by six distinct emotional variants.

Reference Anchor

The base character is generated first and rigidly used as a structural reference for style consistency across all emotions.

Entry Points

The pipeline can be initiated via the CLI utility (hent-ai generate) or through the conversational Discord onboarding flow.

2. Generation Flow

The systematic progression from textual input to the finalized asset bundle.

Step 1: Character Input

  • User provides a text description (e.g., "cute orange cat").
  • System optionally accepts an attached reference image to guide the initial generation.

Step 2: Base Character Generation

  • Generates a neutral-pose character acting as the absolute style anchor.
  • The user is prompted to approve, provide refinement feedback, or regenerate.
  • System exposes the core prompt template used for the anchor.

Step 3: Emotion Variants (×6)

  • Generates six specific emotions sequentially using the approved base as a visual reference.
  • User approves each emotional variant individually.
  • Per-emotion feedback is supported for targeted regeneration without affecting the base.

3. Prompts Reference

The core structural prompts used to interact with the Codex image generation API.

Base Anchor Prompt

{character}, standing in a neutral pose facing forward. Simple clean background, consistent art style, square format, high quality PNG. This is a character reference sheet — no extreme emotion, just the character clearly visible.

Emotion Variant Template

{character}, expressing {emotion}: {descriptor}. Simple clean background, consistent art style, square format, high quality PNG.

Feedback Modifier

When user feedback is provided during regeneration, the following modifier is appended to the prompt:

Additional requirements: {feedback}.

Emotion Descriptors

Emotion Descriptor Binding
happy smiling brightly, giving a thumbs up, celebrating with joy
neutral calm and relaxed, default resting expression, at ease
loyalty saluting attentively, nodding with respect, ready to help
sorry looking apologetic, bowing slightly, sheepish expression
confused tilting head with a puzzled look, question mark above head
focused concentrating intensely, determined expression, working hard

5. Image Requirements

Format

PNG format with transparency support required.

Dimensions

Strictly 1024×1024 square aspect ratio.

File Size

Target payload under 500KB per image.

Background

Simple/clean environment ensuring readability as small Discord thumbnails.

Style Consistency

All 6 emotions must inherently respect the character constraints, physical proportions, and art style defined by the base anchor image.

6. Technical Details

Provider
Codex image generation API (chatgpt.com/backend-api/codex/responses)
Model
gpt-5.4 (default)
SDK Core
god-tibo-imagen for request building and SSE parsing
Fallback Logic
If tool_choice "auto" fails to generate an output, the system retries with "required"
Authentication
Requires local execution of codex login to populate ~/.codex/auth.json
